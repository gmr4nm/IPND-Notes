Lesson 5.1 - Meet Javascript


Adding Javascript to a Page

  • The Easy Way - Put JS directly into your HTML file

  • The Better Way - Write JS in a separate .js file

Use the <script src=“relative path”>(nothing here)</script> tags to define .js in your HTML head. Most browsers will cache the file and won’t have to reload it for each page…this makes your website faster!

Common Syntax

  • Variables: var some_variable = “some_value”;
  • String Methods: [some_string].replace([old], [new]);
  • jQuery: $(selector).action();

JavaScript Object Notation

JSON is a popular and simple format for storing and transferring nested or hierarchal data. It's so popular that most other programming languages have libraries capable of parsing and writing JSON. Internet GET and POST requests frequently pass data in JSON format. JSON allows for objects (or data of other types) to be easily encapsulated within other objects. See the MDN or JSON.org for more details.
Note: you should ‘lint’ your JSON code for errors before you deploy it!

Truthy vs. Falsey

Javascript has a set of values that are equivalent to True and False, but aren’t precisely equal to True/False.

Truthy: true, non-zero numbers, “strings”, objects, arrays, and functions.

Falsy: false, 0, “”, undefined, null, NaN.

Lesson 5.2 - The Power of API’s

API - Application Programming Interface

An API provides a system of building blocks of code, which allows definitions and implementations to vary without compromising the system as a whole. A good API makes it easier to develop a program by providing all key parts requisite for a programmer to combine building blocks from the API with code from multiple sources. The developers then put the pieces together to form novel implementations.

Common Formates API's Use for Communication

  • SOAP (complicated and awful)
  • protocol buffers
  • Thrift (facebook)
  • Plain-text, custom(don't do this)
  • JSON
  • XML

Parsing XML and JSON with Python

XML: from xml.dom import minidom

JSON: import json

json.loads(json_string)

NEVER use eval(json_string)!!! If someone has placed code inside the JSON, this will inject it.

Good Citizenship

  • Use a good user-agent (identify yourself and give a way to get ahold of you if you’re going to make a lot of requests)
  • Rate-limit yourself

Lesson 5.3 - Recursion Programming


Recursive Definitions

Defining something both circularly (in terms of itself) and abstractly. The abstract definition is the the base case (stopping point). The circular definition is the recursive definition.

An example of a recursive definition:

  • Ancestor → Parent (Base)
  • Ancestor → Parent of Ancestor (Recursive)

An Example Defining Factorials

factorial(n) = n * (n - 1) * (n - 2) * (n -3) * … * 1

A recursive definition would look like this:

factorial(0) = 1 (Base Case)

if n > 0

factorial(n) = n * factorial(n -1) (Recursive Case)

Lesson 5.4 - Solving Big Problems

Initially

Sketch out the main concepts or challenges you need to address and which programming principles would be useful for addressing each challenge. Write psuedocode if you have to.

Pacing

Break the problem into small bite sized pieces. Frequently, the biggest challenge isn’t the complexity of any one issue, but the magnitude of dealing with so many things at once. Try to manage this by looking at the problem a little bit at a time.

Use Regression Tests

You want to make sure that as you continue to modify a program, you aren’t breaking functions that you’ve previously built. You can also use regression tests as benchmarks for how fast parts of your program is running. This will save you significant time by preventing you from putting time into bad ideas.
